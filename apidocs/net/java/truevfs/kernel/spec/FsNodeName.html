Class FsNodeName

    Addresses a file system node relative to its mount point.

    Specification

    An node name adds the following syntax constraints to a Uniform Resource Identifier:

    1. The URI must be relative, that is it must not define a scheme component.
    2. The URI must not define an authority component.
    3. The URI must define a path component.
    4. The URI's path must be in normal form, that is its path component must not contain redundant "." and ".." segments.
    5. The URI's path component must not equal ".".
    6. The URI's path component must not equal "..".
    7. The URI's path component must not start with "/".
    8. The URI's path component must not start with "./" (this rule is actually redundant - see #3).
    9. The URI's path component must not start with "../".
    10. The URI's path component must not end with "/".
    11. The URI must not define a fragment component.

    Examples

    Examples for valid node name URIs:

    uri property root property path property query property
    "" true "" null
    "foo" false "foo" null
    "foo/bar" false "foo/bar" null
    "foo?bar" false "foo" "bar"

    Examples for invalid node name URIs:

    URI Issue
    foo:/bar not a relative URI
    //foo/bar authority component defined
    /foo leading slash not allowed
    foo/ trailing slash not allowed
    foo/. not a normalized URI
    foo#bar fragment defined

    Identities

    For any node name e, it's generally true that new FsNodeName(e.getUri()).equals(e).

    Serialization

    This class supports serialization with both ObjectOutputStream and XMLEncoder.
