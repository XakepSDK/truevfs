net.java.truevfs.kernel.spec

Interface FsManager

  • All Superinterfaces:
    FsController.Factory<FsArchiveDriver<? extends FsArchiveEntry>>, FsModel.Factory<FsDriver>
    All Known Implementing Classes:
    FsAbstractManager, FsDecoratingManager, InstrumentingManager, JmxManager

    @ImplementationsShouldExtend(value=FsAbstractManager.class)
public interface FsManager
extends FsModel.Factory<FsDriver>, FsController.Factory<FsArchiveDriver<? extends FsArchiveEntry>>
    A container which creates FsController file system controllers and manages their life cycle.

    Implementations should be thread-safe.

    Author:
    Christian Schlichtherle
    See Also:
    FsController, FsModel

        Filters all managed file system controllers using the given filter and accepts the given visitor to them.
        Parameters:
        filter - the filter for the managed file system controllers. Calls to its Filter.accept(Object) method should terminate quickly without an exception and must not have any side effects on the given controllers!
        visitor - the visitor of the filtered file system controllers. Calls to its Visitor.visit(Object) method may have side effects on the given controllers, e.g. by calling FsController.sync(BitField).
        Returns:
        visitor
        Throws:
        X - at the discretion of the given visitor. Throwing this exception aborts the visiting.

        Returns the thread-safe file system controller for the given mount point. The life cycle of the returned file system controller gets managed by this manager, i.e. it gets remembered for future lookup and synchronization.
        Parameters:
        driver - the composite file system driver which shall get used to create a new file system controller if required.
        mountPoint - the mount point of the file system.
