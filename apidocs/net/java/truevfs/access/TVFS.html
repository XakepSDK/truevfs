      • umount

        public static void umount()
                   throws FsSyncWarningException,
                          FsSyncException
        Commits all pending changes for all (nested) archive files to their respective parent file system, closes their associated target archive file in order to allow access by third parties (e.g. other processes), cleans up any temporary allocated resources (e.g. temporary files) and purges any cached data.

        Note that temporary files may of used even for read-only access to archive files, so calling this operation is essential. However, if the client application never calls this operation, then it gets called by a shutdown hook. The shutdown hook gets removed as soon as this operation gets called in order to leak no memory in dynamic class loader environments.

        Calling this method is equivalent to sync(FsSyncOptions.UMOUNT).

        Throws:
        FsSyncWarningException - if only warning conditions apply. This implies that the respective parent file system has been synchronized with constraints, e.g. if an unclosed archive entry stream gets forcibly closed.
        FsSyncException - if any error conditions apply.
